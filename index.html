PaceVer

PaceVer is a doctoral thesis project that unifies structured pair extraction, bridgeable explanation interfaces, and verification-driven inference-time reasoning for multimodal emotion-cause analysis.

This public site focuses on the dissertation narrative, figures, and current research status. The full research codebase is still maintained privately while the project is under active cleanup and paper-driven refinement.

Thesis framing

Efficiency, completeness, and auditability in one project.

PaceVer treats extraction and explanation as a coupled research problem: first recover reliable emotion-cause links, then expose evidence-grounded explanations that remain measurable.

Overview

A thesis project, not a product landing page

This website presents the research narrative behind the dissertation: why the problem is hard, how the project is decomposed, which parts are already implemented, and where papers, code, and materials can be downloaded.

01

Efficiency

Structured MECPE systems must score many temporally constrained links without turning inference into open-ended generation.

02

Completeness

One-shot thresholding and top-k selection often miss low-confidence but valid long-tail causes that deserve a second look.

03

Auditability

Fluent rationales are not enough. Explanations should point back to explicit evidence and remain parse-back compatible with pair-level evaluation.

How PaceVer is organized

In the thesis, PaceVer is defined as a two-part system: Task I predicts structured emotion-cause links, while Task II returns evidence-grounded explanation objects that can be deterministically mapped back to pair sets.

The extractor module is PaceExt. In the current paper-facing implementation, this appears as the ATDG + DP design and the TRecAM manuscript naming. On top of it, BridgeProt and ReaVer extend the project toward verifiable explanations.

Input Multimodal dialogue

Text, audio, video, speaker structure, temporal precedence.

->
Task I PaceExt

Dialogue-paced pair scoring with a shared temporal horizon.

->
Interface BridgeProt

BL-1 schema, parse-back, parsability, and protocol-aware metrics.

->
Task II ReaVer

Budgeted decoding, verification, and adaptive expansion.

Modules

Three layers of the PaceVer research agenda

The repository tracks both mature contributions and active thesis-stage components. The website distinguishes implemented results from ongoing integration work.

Implemented Extractor

PaceExt

The trained structured extractor that learns a dialogue-paced temporal horizon and outputs a full pair-score matrix under temporal constraints.

  • ATDG generates dialogue-level decay parameters from label-free pace statistics.
  • Dual-Path Temporal Injection aligns kernel smoothing and speaker interaction.
  • A pair-preserving two-stage schedule stabilizes link ranking during multi-task training.
Current results Research outputs Public release status
Protocol Evaluation bridge

BridgeProt

A model-agnostic output and evaluation contract for grounded explanations. BridgeProt defines what a bridgeable explanation should expose, how to parse it back to pair outputs, and which protocol-aware metrics to report.

  • BL-1 minimal schema with explicit evidence pointers.
  • Deterministic parse-back to Pair-F1 and parsability reporting.
  • Strict and salvaged validity metrics for protocol-aware evaluation.
Protocol role Survey framing Availability
Thesis-stage Inference-time reasoning

ReaVer

A verification-driven decoding layer that builds on fixed pair scores from PaceExt. It allocates extra computation only when needed and never replaces the base extractor with an unconstrained generative model.

  • Budgeted candidate selection over structured proposals.
  • Constrained generation of bounded explanation objects.
  • Independent verification and verifier-triggered adaptive expansion.

System view

PaceExt anchors the public experimental results

The current extractor is the most mature component in the project. It operationalizes a dialogue-paced temporal prior through ATDG, aligns recency across modules with DP, and preserves pair decoding through a two-stage schedule.

The thesis then extends this extractor with bridgeable explanation interfaces and verification-driven decoding rather than discarding the structured base.

Resources

Public availability and release path

The dissertation project is evolving, but the core codebase is not yet publicly released. This public repository therefore serves as the stable project homepage and release-status page.

What is public now

  • This GitHub Pages site and its figure assets.
  • A stable high-level summary of the doctoral project.
  • Selected quantitative results and visual evidence from the thesis narrative.

What is not public yet

  • The full research codebase remains in a private working repository.
  • Cleaned public releases of training scripts, configs, and checkpoints are still in preparation.
  • Some manuscripts and thesis materials are being curated before public upload.

Planned release path

The intended long-term setup is: a public project homepage, selected public paper and thesis assets, and a separate cleaned code release once publication and repository cleanup are ready.
